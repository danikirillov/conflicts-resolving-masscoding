# Задание: Порог переупорядочения в менеджере инвентаря

**Файл:** `backend/python/inventory_manager.py`

## Тип конфликта: Выбрать одно значение

### Задание A: Понизить порог переупорядочения
**Расчетное время:** 5 минут

Уменьшите порог переупорядочения для минимизации переизбытка.

**Необходимые изменения:**
- Строка 12: Изменить `self.reorder_threshold = 15` на `self.reorder_threshold = 10`

**Обоснование:** Снизить затраты на складское хранение за счет меньших запасов.

**Инструкции по выполнению:**
```bash
git checkout -b yourname/inventory_manager/a
git add backend/python/inventory_manager.py
git commit -m "Понизить порог переупорядочения до 10"
git push origin yourname/inventory_manager/a
```

---

### Задание B: Повысить порог переупорядочения
**Расчетное время:** 5 минут

Увеличьте порог переупорядочения для предотвращения дефицита.

**Необходимые изменения:**
- Строка 12: Изменить `self.reorder_threshold = 15` на `self.reorder_threshold = 20`

**Обоснование:** Избежать потерянных продаж из-за отсутствия товара на складе.

**Инструкции по выполнению:**
```bash
git checkout -b yourname/inventory_manager/b
git add backend/python/inventory_manager.py
git commit -m "Повысить порог переупорядочения до 20"
git push origin yourname/inventory_manager/b
```

---

## Ожидаемый конфликт
Оба задания изменяют порог переупорядочения с конкурирующими приоритетами (стоимость vs. доступность).

## Стратегия разрешения
**Выберите одно** на основе:
- Исторической частоты дефицита
- Затрат на склад
- Надежности поставщиков
- Прогнозов продаж
