# Задание: Порог перезаказа в менеджере инвентаря

**Файл:** `backend/python/inventory_manager.py`

## Тип конфликта: Выбрать одно значение

### Задание A: Понизить порог перезаказа
**Расчетное время:** 5 минут

Уменьшите порог перезаказа для минимизации переизбытка.

**Необходимые изменения:**
- Изменить `self.reorder_threshold = 15` на `self.reorder_threshold = 10`

**Обоснование:** Снизить затраты на складское хранение за счет меньших запасов.

**Инструкции по выполнению:**
```bash
git checkout -b <твой ник или имя>/inventory_manager/a
git add backend/python/inventory_manager.py
git commit -m "Понизить порог перезаказа до 10"
git push origin <твой ник или имя>/inventory_manager/a
```

---

### Задание B: Повысить порог перезаказа
**Расчетное время:** 5 минут

Увеличьте порог перезаказа для предотвращения дефицита.

**Необходимые изменения:**
- Изменить `self.reorder_threshold = 15` на `self.reorder_threshold = 20`

**Обоснование:** Избежать потерянных продаж из-за отсутствия товара на складе.

**Инструкции по выполнению:**
```bash
git checkout -b <твой ник или имя>/inventory_manager/b
git add backend/python/inventory_manager.py
git commit -m "Повысить порог перезаказа до 20"
git push origin <твой ник или имя>/inventory_manager/b
```

---

## Ожидаемый конфликт
Оба задания изменяют порог перезаказа с конкурирующими приоритетами (стоимость vs. доступность).

## Стратегия разрешения
**Выберите одно** на основе:
- Исторической частоты дефицита
- Затрат на склад
- Надежности поставщиков
- Прогнозов продаж
